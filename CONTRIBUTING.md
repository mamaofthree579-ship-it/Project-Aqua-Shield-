# Contributing to ProjectAquaShield

## Thank you!
We welcome contributions of all kinds: documentation, translations, tests, field reports, designs, and outreach materials.

## How to get started
1. Fork the repo and create a branch: `git checkout -b my-feature`
2. Run tests (see `documentation/tests.md` for test protocols for hardware & filters).
3. Open a Pull Request describing your change and link any test data.

## Issue templates
- Use bug reports for problems with designs, documentation, or tests.
- Use feature requests for new designs, regional guides, or outreach campaigns.

## Region-specific contributions
When adding a regional guide, include:
- Local water contamination profile (if available)
- Regulatory considerations
- Locally-available materials and suppliers
- Language translation

## Translation workflow
- Add new language folder under `documentation/`
- Mark the PR with `i18n` label and assign a reviewer fluent in the language

## Design & test protocols
All designs must include:
- CAD/source files
- Bill of Materials (BOM)
- Testing logs and sample methodology (CSV under `data/`)

## Code of Conduct
By contributing you agree to abide by `CODE_OF_CONDUCT.md`.
